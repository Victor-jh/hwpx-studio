## 3. 기안문 구조

공문서(기안문)는 **두문 → 본문 → 결문** 3단 구조.

### 3.1 두문 (Header)

| 요소 | 설명 |
|------|------|
| 기관명 | 발신 기관의 공식 명칭 |
| 수신 | 수신 기관 또는 "수신자 참조" |
| (경유) | 경유 기관이 있는 경우에만 |

### 3.2 본문 (Body)

| 요소 | 설명 |
|------|------|
| 제목 | 문서 내용을 요약한 제목 (간결하게) |
| 본문 내용 | 왼쪽 정렬, 첫 글자부터 시작 (2025 개정) |
| 항목 기호 | 1단: 1. 2. 3. → 2단: 가. 나. 다. → 3단: 1) 2) 3) → 4단: 가) 나) 다) |
| 붙임 | 첨부 파일 목록 (본문 끝에) |

### 3.3 결문 (Footer)

| 요소 | 설명 |
|------|------|
| 발신명의 | 기관장 직함 + 성명 |
| 기안자/검토자/결재자 | 직급 + 서명 |
| 협조자 | 필요 시 |
| 시행/접수 | 문서번호-일련번호 (시행일자/접수일자) |
| 우편번호/주소/전화/팩스 | 기관 연락처 |
| 공개구분 | 공개, 부분공개, 비공개 |

---

## 4. 항목 기호 체계

```
1.  (제1단계)
  가.  (제2단계)
    1)  (제3단계)
      가)  (제4단계)
        (1)  (제5단계)
          (가)  (제6단계)
            ①  (제7단계)
              ㄱ.  (제8단계)
```

## 5. hwpx-studio 기안문 JSON 예시

```json
{
  "blocks": [
    {"type": "heading", "level": 1, "text": "○○ 업무 추진 계획"},
    {"type": "empty"},
    {"type": "numbered", "text": "추진 배경"},
    {"type": "paragraph", "text": "「○○법」 제00조에 따라 추진함."},
    {"type": "empty"},
    {"type": "numbered", "text": "추진 내용"},
    {"type": "indent", "text": "가. 1차 계획: 2025. 1. ~ 6."},
    {"type": "indent", "text": "나. 2차 계획: 2025. 7. ~ 12."},
    {"type": "empty"},
    {"type": "numbered", "text": "기대 효과"},
    {"type": "paragraph", "text": "업무 효율 30% 향상이 기대됨."},
    {"type": "empty"},
    {"type": "paragraph", "text": "붙임  1. 세부 추진 일정 1부.  끝."}
  ]
}
```

**생성 명령:**
```bash
hwpx-create input.json --style gonmun -o 기안문.hwpx
```

## 6. 공문서 작성 원칙

| 원칙 | 설명 |
|------|------|
| 정확성 | 내용이 정확하고 오류가 없을 것 |
| 용이성 | 읽기 쉽고 이해하기 쉬울 것 |
| 간결성 | 1건 1매 원칙, 불필요한 내용 배제 |
| 경제성 | 불필요한 여백/페이지 낭비 방지 |
