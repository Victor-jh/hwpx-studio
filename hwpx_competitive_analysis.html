hwpx-studio SWOT 분석

💪 Strength: AI Agent 네이티브 설계

처음부터 Claude Code/Cursor/Codex 에이전트를 위해 설계됨. JSON 블록 → HWPX → JSON 라운드트립이 에이전트 워크플로우에 최적화. 다른 어떤 프로젝트도 이 수준의 AI 통합을 제공하지 않음.

💪 Strength: 35개 블록 타입 + KCUP

KCUP 16개 전용 블록 타입은 세계 유일. 한국 공공기관 보고서 포맷을 네이티브로 지원하는 유일한 도구. python-hwpx조차 KCUP을 모름.

💪 Strength: 서식 정밀도 95점

PropertyRegistry의 동적 charPr/paraPr/borderFill 할당은 python-hwpx의 dataclass 매핑보다 실전 문서 재현에서 더 정밀함. 레퍼런스 기반 복원 + page_guard 조합은 독보적.

💪 Strength: 라운드트립 100%

HWPX → JSON → HWPX → JSON에서 264블록/16타입 100% 일치 검증 완료. 다른 프로젝트 중 이 수준의 라운드트립을 입증한 곳은 없음.

⚠️ Weakness: PyPI 미배포

pip install이 불가. python-hwpx, pypandoc-hwpx는 모두 PyPI에 등록. 패키지 매니저 접근성에서 열세. 잠재 사용자가 발견하기 어려움.

⚠️ Weakness: 자동화 테스트 부재

pytest 기반 테스트 스위트가 없음. python-hwpx는 23개 테스트 + 60% 커버리지. 코드 변경 시 회귀 위험이 높고, CI/CD 파이프라인 없음.

⚠️ Weakness: Private 리포

GitHub Stars 0 (비공개). 커뮤니티 기여·피드백·발견 불가. hwpxlib(171⭐), pyhwp(293⭐), pypandoc-hwpx(150⭐)에 비해 인지도 제로.

⚠️ Weakness: 다중 섹션 read/edit 미지원

read_document.py와 edit_document.py가 section0.xml만 처리. 긴 문서에서 section1, section2가 있으면 누락됨.

🚀 Opportunity: 공개 전환 + PyPI 등록

공개 + PyPI 배포 시 한국 HWPX 생태계에서 유일한 "AI Agent 네이티브 + KCUP 지원" 라이브러리로 포지셔닝 가능. LobeHub/MCP Marketplace 등록도 가능.

🚀 Opportunity: MCP 서버 구현

python-hwpx가 이미 MCP 서버(mcp-hwpx)를 제공. hwpx-studio도 MCP 서버를 만들면 Claude/Gemini/ChatGPT 등 모든 AI에서 사용 가능.

🚀 Opportunity: pypandoc-hwpx 연동

pypandoc-hwpx(150⭐)는 MD/DOCX→HWPX 변환만 가능. hwpx-studio의 read/edit 기능과 결합하면 양방향 파이프라인 완성.

🔥 Threat: python-hwpx의 급성장

v2.8.3까지 빠르게 진화 중. MCP 서버, PyPI, Sphinx 문서, 23개 테스트. 범용 라이브러리로서 AI 스킬 영역까지 확장 가능성. 가장 직접적인 경쟁자.

🔥 Threat: 한컴 공식 도구 등장 가능성

한컴이 hancom-io/hwpx-owpml-model을 공개. 향후 공식 Python SDK를 발표하면 서드파티 전체가 타격.
